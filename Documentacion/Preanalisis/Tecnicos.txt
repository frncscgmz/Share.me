Entorno de programación

ActiveState Komodo IDE
Komodo IDE  es un editor de texto para lenguajes de programación dinámicos, tiene características  como el rango completo de lenguajes soportados (Perl, PHP, Python, Ruby, Tcl, SQL, Smarty) y plataformas (Linux, Mac OS X y Windows). Puede editar archivos remotamente por FTP. Además soporta lenguajes del lado del cliente como Javascript, CSS, HTML y XML.
Fue desarrollado por programadores que necesitan un editor multi-lenguaje con ampliada funcionalidad. Decidimos utilizar este IDE ya que cuenta con modos de integrarse con Git para manejar los repositorios y el control de versiones, además de todas las poderosas herramientas de edición con las que cuenta.
Lenguaje de programación


PHP
PHP es un lenguaje de programación del lado de servidor de uso general diseñado para el desarrollo web para producir páginas web dinámicas. Para este fin, el código PHP se integra con el documento fuente HTML e interpretado por un servidor web con un módulo de procesamiento PHP, el cual genera la página web. PHP puede ser empleado en la mayoría de los servidores web como un intérprete único, en casi cualquier sistema operativo y plataforma sin costo alguno.
Hemos decidió utilizar este lenguaje ya que cubre todas nuestras necesidades para realizar una página web e interactuar con una base de datos. Además de que es gratis, existe una gran cantidad de soporte en línea.


Software para control de versiones y manejo de repositorios.

GitHub
Se trata de un servicio web para hospedar proyectos de desarrollo de software utilizando el sistema de control de revisiones Git. GitHub ofrece cuentas gratis y comerciales para proyectos open source.
El sitio provee funcionalidades en red social tales como feeds, seguidores y grafos de red para mostrar como los desarrolladores trabajan en las versiones de sus repositorios. 
Algunas de sus características incluyen:
•	 Hospedaje de código
GitHub mantiene el código público y privado siempre listo, seguro y respaldado. De esta forma evita que los desarrolladores mantengan todo su código en el disco duro y sin respaldo.
•	Manejo de equipos por organizaciones
Ya sea que se esté manejando un proyecto de open source o un proyecto grande para una compañía, las organizaciones simplifican el manejo de equipos. Con los equipos se le puede dar a los desarrolladores el poder que necesiten, desde la habilidad de crear proyectos en nombre de la organización hasta dar accesos de solo lectura en proyectos existentes.
No existe límite para el número de equipos que se puedan crear o miembros los cuales puedan accesar.
•	Rastreador de problemas
Cada repositorio en GitHub viene con rastreador de problemas, para que de esta forma los desarrolladores puedan mantenerse al pendiente de los errores y se puedan enfocar también en el desarrollo. Tareas, etiquetas y metas ayudan a manejar proyectos grandes mientras los atajos de teclado simplifican la creación de listas de quehacer o una lista de bugs en el proyecto.
•	Revisión de Código
Las peticiones de extracción son discusiones en vivo que simplifican el proceso de verificación, reseña y manejo de cambios de código con el equipo. Cada petición toma en cuanto no solo lo que se quiere extraer sino en donde se quiere realizar ciertos cambios. Desde ahí el equipo puede discutir los cambios en general o por partes individuales.


Git
Git es un sistema de distribuido de control de revisiones con énfasis en la velocidad. Cada directorio en Git es un repositorio con historial completo y con capacidades de seguimiento de versiones, sin depender de acceso al servidor central.
Algunas de sus características son:
•	Soporte sólido para desarrollo no lineal
•	Desarrollo distribuido
•	Compatibilidad con sistemas y protocolos ya existentes
•	Manejo eficiente de proyectos grandes
•	Diseño basado en herramientas
•	Entre otros
El diseño de Git es una síntesis de las experiencias de Linus Torvalds con Linux al mantener proyectos grandes de desarrollo distribuido, junto con su conocimiento íntimo del rendimiento sistema de archivos adquirido por el mismo proyecto y la necesidad urgente de producir un sistema en poco tiempo.
